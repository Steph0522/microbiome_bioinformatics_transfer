TEMA 5 TEMA : Predicción funcional con PICRUSt

5.1 ¿Qué es PICRUSt2?

PICRUSt2 (Phylogenetic Investigation of Communities by Reconstruction of Unobserved States/Investigación Filogenética de Comunidades Mediante Reconstrucción de Estados No Observados) es un software utilizado para predecir las funciones de una comunidad de organismos y sus abundancias basado en secuencias de genes marcadores.

En metabarcoding, usualmente nos referimos a “función” o “genes funcionales” como ortólogos de KEGG y enzimas, pero las predicciones pueden realizarse para cualquier rasgo arbitrario. Las predicciones típicamente se basan en datos de secuenciación de genes 16S rRNA, pero recientemente también se pueden utilizar otros genes marcadores .

5.2 PICRUSt2

PICRUSt2 es la segunda versión de PICRUSt ó PICRUSt1 (su primera versión), el cual mejoró en muchos aspectos:

Tiene una base de datos actualizada (v1 no actualizada desde 2013) y más grande (>10x) de géneros, familias y genomas de referencia

Permite usar OTUs o ASVs (anteriormente solo OTUs closed-reference contra la base de datos de greengenes)

Es más preciso y puede usarse con otros marcadores como 18S rRNA e ITS

Permite la adición de bases de datos de referencia personalizadas

Permite la generación de predicciones de ontología de MetaCyc que son comparables con las de metagenómica shotgun

La inferencia de las abundancias de las vías ahora se basa en MinPath, lo que hace que estas predicciones sean más rigurosas

5.3 Fundamento de PICRUSt2

PICRUSt2 integra herramientas y programas de código abierto existentes para predecir genomas a partir de secuencias de genes de 16S rRNA muestreados ambientalmente.

Las features (ASVs u OTUs) se colocan en un árbol de referencia, que se utiliza como base para las predicciones funcionales. Este árbol de referencia contiene 20,000 genes completos de 16S rRNA de bacterias y arqueas de genomas en la base de datos Integrated Microbial Genome (IMG).

Las predicciones de PICRUSt2 basadas en varias bases de datos de familias de genes están respaldadas de manera predeterminada, incluidos los ortólogos de la Enciclopedia de Genes y Genomas de Kyoto (KEGG) (KOs) y los números de la Comisión Enzimática (EC).

5.4 Flujo de trabajo de PICRUSt2

Primer paso: Con HMMER se alinean las ASVs con los genomas de referencia, implementa métodos que utilizan modelos probabilísticos llamados modelos y perfiles ocultos de Markov.

Segundo paso: EPA-ng o SEPP se usan para determinar la posición óptima de las ASVs colocadas en una filogenia de referencia, y GAPPA para generar un nuevo árbol que incorpore las ASVs. Dando como resultado un árbol filogenético que contiene tanto genomas de referencia como organismos muestreados ambientalmente.

Tercer paso: El arbol filogenético obtenido se utiliza para predecir el número de copias de genes individuales para cada ASV (copias 16S, EC y KO). Para esto se usa el paquete de R castor que predice los estados ocultos. Se pueden usar bases de datos personalizadas.

Cuarto paso: Las secuencias variantes del gen (ASVs) se corrigen en PICRUSt2 según el número de copias del 16S rRNA y luego se multiplican por sus predicciones funcionales para producir un metagenoma predicho. PICRUSt2 también proporciona la contribución de cada función predicha de las ASVs, lo que permite realizar análisis estadísticos informados por la taxonomía.

Quinto paso: Por último, las abundancias de las vías metabólicas se infieren usando MinPath (MetaCyc) en función de mapeos de vías estructuradas, que son más conservadoras que el enfoque de ‘conjunto de genes’ utilizado en PICRUSt1. Las abundancias y coberturas se calculan basado en ECs.

5.5 Limitaciones de PICRUSt2

  • Las predicciones están limitadas en cómo las secuencias de tu estudio puedan ser ubicadas en el árbol de referencia por lo que no puede ser tan reproducible con otro conjunto de datos.

  • Ciertos entornos están mejor representados por genomas de referencia que otros. Por ejemplo, se espera que PICRUSt2 funcione mejor con secuencias del intestino humano, por lo que crear una base de datos de referencia personalizada de genomas específicos para tu entorno de interés puede ser una opción viable.

  • Por defecto, las secuencias de entrada con valores de NSTI (nearest sequence taxon index) superiores a 2 se excluyen del análisis. Esto podría afectar potencialmente algunas muestras mucho más que otras.

  • Las asignaciones de PICRUSt2 a funciones de alto nivel, como vías metabólicas, dependen completamente del archivo de asignación utilizado. Por lo tanto, vale la pena verificar cuidadosamente las anotaciones para asegurarse de que sean razonables para tu sistema y filtrar cualquier vía irrelevante.

  • Las predicciones están limitadas a la porción del metagenoma completo que es contribuida por los organismos amplificados por tus primers.

5.6 Asignación funcional con PICRUSt2 en QIIME2

5.6.1 PICRUSt2 en qiime2

Lo primero que debemos hacer es activar el ambiente de conda de qiime2 (qiime2-2024.5 o más actual recomendado):

conda activate qiime2-2024.5

Luego vamos a instalar el plugin con conda:

conda install q2-picrust2=2024.5 \
-c conda-forge \
-c bioconda \
-c gavinmdouglas

En caso de incompatibilidades, se puede instalar manualmente. Ver las instrucciones en este link:

Nota: Ten en cuenta que muchas de las características disponibles en la versión independiente aún no están implementadas en el plugin de QIIME2.

Vamos a revisar los parámetros y requerimientos de la función:

qiime picrust2 full-pipeline --help

Los archivos de entrada requeridos son --i-table y --i-seq, que son los archivos de tipo FeatureTable[Frequency] y FeatureData[Sequence]. La tabla contiene las abundancias de las ASVs y el otro archivo las secuencias representativas de estas ASVs. Hay varios parámetros que se pueden modificar según sea el caso.

Algunas opciones:

--p-placement-tool: si hacer el alineamiento con epa-ng ó sepp.

--p-max-nsti: especifica qué tan distante debe estar una secuencia en el árbol de referencia antes de que sea excluida. El valor predeterminado es 2. Para los conjuntos de datos ambientales, una mayor proporción de ASVs puede ser excluida en función de este valor predeterminado.

--p-threads: indica el número de CPUs a usar.

Ahora, descargaremos la tabla y secuencias a utilizar en este ejemplo:

wget --no-check-certificate 'https://docs.google.com/uc?export=download&id=1Gyu2sXQeBY2gN1aopyeFPi0DmysGCq6W' -O table_picrust2.qza
wget --no-check-certificate 'https://docs.google.com/uc?export=download&id=1a9zEEhDeJPk2jZbOBsuJR_XeVBP-3agN' -O seqs_picrust2.qza

El código para ejecutarlo es:

qiime picrust2 full-pipeline \
   --i-table table_picrust2.qza \
   --i-seq seqs_picrust2.qza \
   --output-dir q2-picrust2_output \
   --p-threads 6 \
   --verbose

Los archivos de salidas son:

  • ec_metagenome.qza: predicciones de EC (los números EC en las filas y las muestras en las columnas)

  • ko_metagenome.qza: predicciones de KO’s

  • pathway_abundance.qza: predicciones de rutas de MetaCyc

Todos son artefactos con tipo semántico FeatureTable[Frequency], así que, podemos trabajar dentro de qiime2 como una tabla de abundancias.

Para generar los visualizadores de las tablas usaremos los siguientes códigos:

qiime feature-table summarize \
   --i-table q2-picrust2_output/pathway_abundance.qza \
   --o-visualization q2-picrust2_output/pathway_abundance.qzv
qiime feature-table summarize \
   --i-table q2-picrust2_output/ec_metagenome.qza \
   --o-visualization q2-picrust2_output/ec_metagenome.qzv
qiime feature-table summarize \
   --i-table q2-picrust2_output/ko_metagenome.qza \
   --o-visualization q2-picrust2_output/ko_metagenome.qzv

Warning! Ten en cuenta que estos archivos no están en unidades de abundancia relativa y en su lugar, es la suma de la abundancia funcional predicha contribuida por cada ASV y multiplicada por la abundancia (el número de lecturas de entrada) de cada ASV.

5.7 Análisis diferencial de abundancia: ANCOM-BC

El artículo de ANCOM-BC (Analysis of Compositions of Microbiomes with Bias Correction) pueden verlo aquí

Luego de haber obtenidos los genes y rutas funcionales nos interesa saber qué rutas o genes están variando de manera diferencial entre una sección y otra. Es decir, cuál se enriquece o incrementa en una sección en comparación con la otra. Para esto utilzaremos la función ancombc del plugin composition.

Para esta parte también vamos a necesitar la tabla de los metadatos de las muestras:

wget --no-check-certificate 'https://docs.google.com/uc?export=download&id=1yUrJvOj-ffdOQ3bunQPKaDx4LjJOCFY9' -O metadata.transf.2600.txt

El código para hacer el análisis ancom-bc es el siguiente:

 qiime composition ancombc\
   --i-table q2-picrust2_output/pathway_abundance.qza\
   --m-metadata-file  metadata.transf.2600.txt\
   --p-formula Seccion\
   --o-differentials  differentials_pathways.qza

Para visualizar los resultados:

qiime composition da-barplot\
  --i-data differentials_pathways.qza\
  --o-visualization differentials_pathways.qzv

qiime2 view

Este ejercicio lo hicimos para las rutas o pathways. ¿Qué tenemos que hacer para calcularlo para los EC o genes funcionales?

Solución

Si queremos exportar los resultados de este análisis

qiime tools export\
  --input-path differentials_pathways.qza\
  --output-path exported_picrust2

5.8 Exportando y agregando descripciones a las rutas

Vamos a hacer este ejemplo solo para las rutas, pero podemos aplicarla en todos los archivos de salida.

Primero vamos a exportar la tabla de frecuencias de rutas:

qiime tools export\
  --input-path q2-picrust2_output/pathway_abundance.qza\
  --output-path exported_picrust2

Luego, vamos a convertir la tabla de biom a texto con el siguiente código:

biom convert -i exported_picrust2/feature-table.biom\
  -o exported_picrust2/pathways.txt\
  --to-tsv

Descargamos script de R:

wget --no-check-certificate 'https://docs.google.com/uc?export=download&id=1DPDFttTnNH9EbfHP33ssM4k9pTXbkzTK' -O pathways_descriptions.R

Con el siguiente script de R vamos a agregar las descripciones:

Rscript pathways_descriptions.R\
  -i exported_picrust2/pathways.txt\
  -o exported_picrust2/pathways_description.txt
